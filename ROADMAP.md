# Roadmap

This roadmap describes a practical adoption path for teams implementing this
threat-modeling approach.

## First 30 days

- Align stakeholders on scope, goals, and success criteria.
- Run one pilot threat-modeling session on a high-value agent workflow.
- Score top threats with ARS and identify immediate control gaps.
- Create an initial risk register with owners and due dates.

## Days 31 to 60

- Expand reviews to additional agent workflows and tool integrations.
- Add threat modeling as a required input for architecture changes.
- Define standard evidence requirements for threat submissions.
- Track remediation progress and residual risks in recurring reviews.

## Days 61 to 90

- Operationalize review cadence tied to release and change-management cycles.
- Build internal metrics (high-tier threats open, mean remediation time, control coverage).
- Establish formal ownership boundaries across product, security, and operations.
- Publish periodic taxonomy update notes and lessons learned.

## Future enhancements

- Add sector-specific examples (healthcare, finance, critical infrastructure).
- Add deeper crosswalks to additional governance and assurance frameworks.
- Add optional automation helpers for risk register generation.
