# Assumptions

This repository is designed around the following assumptions:

- The target system is a deployed agentic AI workload, not only an offline model.
- The system can execute actions through tools, APIs, or integrated services.
- Security teams need practical guidance they can apply during real design reviews.
- Risk prioritization requires both technical and operational context.
- Organizations using this material maintain their own policy, legal, and compliance controls.

## Threat-modeling assumptions

- Threats are evaluated relative to explicit trust boundaries.
- Agent autonomy meaningfully affects risk severity.
- Detectability is a first-class scoring concern, not an afterthought.
- Residual risk is documented with an owner and review date.

## Contribution assumptions

- Contributions should optimize for clarity and practical applicability.
- New threat patterns should include realistic preconditions.
- Major guidance changes should be discussed in issues or pull requests.
